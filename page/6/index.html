语义版本控制（SemVer）

 Lruihao 发布于 2022-08-13 收录于 程序猿的自我修养
摘要

版本格式：MAJOR.MINOR.PATCH，版本号递增规则如下：

  1. MAJOR: 主版本号，当你做了不兼容的 API 修改
  2. MINOR: 次版本号，当你做了向下兼容的功能性新增
  3. PATCH: 修订号，当你做了向下兼容的问题修正

先行版本号及版本编译信息可以加到 MAJOR.MINOR.PATCH 的后面，作为延伸。
